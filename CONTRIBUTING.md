## How to Contribute

### Reporting Bugs
If you find a bug, please create a new issue on GitLab. Be sure to include:
*   Steps to reproduce
*   Expected behavior
*   Actual behavior
*   Logs or screenshots if applicable

### Suggesting Enhancements
We welcome architectural improvements! Please open an issue to discuss the design before submitting a large Merge Request.

### Pull Request Process
1.  Fork the repository.
2.  Create a new branch (`git checkout -b feature/AmazingFeature`).
3.  Commit your changes (`git commit -m 'Add some AmazingFeature'`).
4.  Push to the branch (`git push origin feature/AmazingFeature`).
5.  Open a Merge Request against the `main` branch.

### Coding Standards
*   **Java:** Follow standard Spring Boot conventions. Use shared components from `common-*` modules.
*   **Angular:** Use standalone components and Signals.
*   **Documentation:** Update README or `docs/` folder if you change architectural flows.
